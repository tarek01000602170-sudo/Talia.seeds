# نظام إدارة المبيعات — نسخة Firebase

تطبيق ويب كامل (PWA) قابل للتثبيت على الموبايل والكمبيوتر، يدعم العربي والإنجليزي، البيانات على Firebase.

---

## محتويات الحزمة

| الملف | الوظيفة |
|------|---------|
| `index.html` | التطبيق الكامل |
| `manifest.json` | إعدادات تثبيت التطبيق على الموبايل |
| `sw.js` | يسمح بتشغيل التطبيق أوفلاين |
| `icons/` | أيقونات التطبيق |
| `firestore.rules` | قواعد أمان قاعدة البيانات |
| `storage.rules` | قواعد أمان رفع الصور |

---

## خطوات التشغيل (مرة واحدة فقط)

### 1) إعداد Firebase

ادخل [Firebase Console](https://console.firebase.google.com) → مشروعك `sedy-5ca6a`:

**أ. فعّل Authentication:**
- Authentication → Sign-in method → فعّل **Email/Password**
- Users → Add user → اعمل حساب أدمن (إيميل وباسورد)

**ب. أنشئ Firestore Database:**
- Firestore Database → Create database → ابدأ في وضع Production
- Rules → انسخ محتوى `firestore.rules` والصقه → Publish

**ج. فعّل Storage:**
- Storage → Get started → اقبل الإعدادات الافتراضية
- Rules → انسخ محتوى `storage.rules` والصقه → Publish

### 2) رفع التطبيق على GitHub Pages

1. أنشئ repository جديد على GitHub
2. ارفع كل الملفات (index.html, manifest.json, sw.js, icons/)
3. Settings → Pages → Branch: main → Save
4. التطبيق هيشتغل على: `https://USERNAME.github.io/REPO/`

### 3) أول دخول

افتح الرابط، اضغط "تسجيل الدخول" واستخدم الإيميل والباسورد اللي عملته في الخطوة 1.أ.

عند أول استخدام، اضغط زرار **"إضافة بيانات تجريبية"** علشان يضيف منتجات وعملاء وبائعين كبداية.

## تثبيت التطبيق على الموبايل (Android)

### الطريقة الأسهل: تثبيت كـ PWA
1. افتح رابط التطبيق على Chrome في الموبايل
2. هيظهر زر "إضافة إلى الشاشة الرئيسية" (Add to Home Screen)
3. التطبيق هيظهر زي أي تطبيق عادي

### للحصول على ملف APK حقيقي:
1. افتح [PWABuilder.com](https://www.pwabuilder.com)
2. ألصق رابط تطبيقك على GitHub Pages
3. اضغط Build → Android → Download APK
4. هتحصل على ملف APK جاهز للتنزيل والتثبيت

### للحصول على ملف على Google Play:
PWABuilder بيديك Android Package Bundle جاهز للرفع على Google Play Console (يحتاج حساب مطور بـ 25$ مرة واحدة).

## ملاحظات مهمة عن الأمان

- **منع F12 موجود في التطبيق** لكنه مجرد حاجز للمستخدم العادي. أي مبرمج يقدر يتجاوزه.
- **الأمان الحقيقي = قواعد Firestore و Storage** اللي في الملفات. هي اللي بتحمي بياناتك من أي حد يحاول يعدل في الكود.
- **لا تشيل قواعد الأمان أبداً** ولا تخليها مفتوحة للجميع.

## الميزات المتوفرة

**لوحة التحكم:**
- إحصائيات شاملة + مخزون منخفض + رسم بياني للمبيعات (آخر 14 يوم)
- رسم بياني لأفضل المنتجات + جدول أفضل البائعين + سجل النشاط

**المنتجات:**
- فئات (إلكترونيات، ملابس، إلخ) + بحث بالاسم/الكود
- استيراد وتصدير Excel (الأعمدة: code, name, price, profit, weight, stock, category, description)
- صور متعددة، أكواد، أوزان، أرباح

**العملاء:**
- استيراد وتصدير Excel (code, name, phone, phone2, governorate, address, status)
- تليفونين، محافظة، حالة العميل

**البائعين:**
- إنشاء حساب دخول لكل بائع (إيميل + باسورد)
- البائع لما يدخل بحسابه يشوف صفحته الخاصة بس

**المبيعات:**
- أكتر من منتج في الأوردر الواحد
- بحث بالحالة + الضغط على الحالة يفتح تفاصيل الأوردر
- تصدير كل الأوردرات Excel

**الفواتير:** بحث وفلترة بالحالة

**الشحن:**
- إدارة شركات الشحن
- مصفوفة أسعار (محافظة × توصيل منزل/مكتب × وزن)
- بحث بالشركة والحالة، حساب السعر تلقائياً

**المخزون:** عرض المتاح والمحجوز في الأوردرات

**واتساب:** فلتر العملاء حسب البائع وإرسال رسائل

**شاشة التغليف:**
- طباعة استيكرات باركود بمقاس مخصص
- عرض تفاصيل الأوردرات الكاملة + طباعة عنوان الشحن

**صفحة بائع موبايل:**
- صور المنتجات في شبكة، اضغط الصورة = +1، اضغط الاسم = -1
- بحث بتليفون العميل (يجيب بياناته أو يضيف جديد)

**المتجر العام:**
- فئات في الأعلى للفلترة
- قائمة منسدلة للكمية
- صور متعددة بعرض جاليري

**عام:**
- عربي/إنجليزي مع تبديل لحظي
- موبايل وكمبيوتر، قابل للتثبيت كـ PWA
- بيانات حية متزامنة (Firestore Real-time)
- ٣ صلاحيات: مدير، بائع، تغليف

## مشاكل شائعة

**التطبيق يعرض "Permission denied":**
لازم تنشر قواعد Firestore (الخطوة 1.ب أعلاه)

**لا أستطيع تسجيل الدخول:**
تأكد إن Email/Password مفعّل في Firebase Auth وإن المستخدم موجود

**صور المنتجات لا ترفع:**
تأكد من إعداد Storage ونشر `storage.rules`
